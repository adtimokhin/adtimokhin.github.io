News. Entity and Validation

Let's try to use all of our knowledge to create this new entity for news. We will also add service for News and will create a new validator for News.

Creating News entity

This entity will be used to store news on our website. In my opinion, every News objects will need:

Id
Headline
News text (general text of the actual news) article.

We will expand this list of attributes later, but for now this is more than enough.

Do the following:

    • Create a new News class with required fields, getters and setters, constructors
    • Create a new table that matches our News entity

My table looks like this:

I assigned news_headline and news_text columns to be notnull, so that every news item we create has a headline and the news text.

My News class looks like this:


@Entity

@Table (name = "news")

public class News {


    @Id

    @GeneratedValue (strategy = GenerationType.IDENTITY)

    private int id;


    @Column (name = "news_headline")

    private String headline;


    @Column (name = "news_text")

    private String text;


    // Constructors


    public News () {

    }


    public News (String headline, String text) {

        this.headline= headline;

        this.text= text;

    }


    // Getters and setters


    public int getId () {

        return id;

    }


    public void setId (int id) {

        this.id= id;

    }


    public String getHeadline () {

        return headline;

    }


    public void setHeadline (String headline) {

        this.headline= headline;

    }


    public String getText () {

        return text;

    }


    public void setText (String text) {

        this.text= text;

    }


}

Here you see I used a new annotation @Column. This annotation is used to specify to what table column this attribute will be mapped. This is especially useful when your column name and name of an attribute in your entity class do not match.

But what if our user knows what article he wants to view? What if he is so in love with what he has seen in that article that he has found out id of that article? Well, we can help him with that.

Instead of passing a single parameter of some simple type (including String) in a form, we use the URL itself to pass that data. There are two ways of doing it.

1) We can add this parameter to our URL directly like this:

NewsController.class


    // ...


    @GetMapping ("/get/news/{id}")

    public String getANewsArticle (@PathVariable (name = "id") int id, Model model) {

       // do some work here ...

    }


    // ...


@PathVariable annotation tells Spring that a variable that is passed inside {} in the URL is actually some useful data that we want to be able to access.

Example:

From URL like this:

localhost:8080:/news/view/2

Spring will extract '2' as an integer that we would be able to use.


2) We can add parameters to URL in this form:

NewsController.class


    // ...


    @GetMapping ("/get/news")

    public String getANewsArticle (@RequestParam (name = "id") int id, Model model) {

       // do some work here ...

    }


    // ...


@RequestParam is an annotation that will extract data that we pass inside URL in the format like this:

localhost:8080:/news/view?id=2

You can specify if you need that parameter to be passed in order to carry the request or not. If you select that it is not required, then you will just get an unmeaningful value for that variable. This is true for both @PathVariable and @RequestParam annotations.

You are not limited to passing a single variable through your URL path if you use the second method. You can choose to have many variables there.

You are totally welcome to select any approach you like to pass id of our news to the getANewsArticle() method.

But anyhow, let us make that getANewsArticle() method do some useful work.

    • Create getANewsArticle() method in NewsController. You may want to update your NewsService to have a method that will return a single News by id.

Adding News to the website

Now it is time to introduce Spring Validation.

We will start of by adding a FTL page for creating News entities. I will now show how it looks and then we will go over what is going on in it.

createNews.ftl

This is a createNews.ftl page. And it has some new things that we didn’t see in FTL before.

First of all let’s look at the top of the page:

createNews.ftl

This line is used to create an object that will have the functionality of a library that we pass in the brackets. This library http://www.springframework.org/tags/forms Is a library that expands the ways how we can use Spring to interact with forms. The most important part is that now we can send and receive data to and from this page as an entity.

Now let's shift our attention to this odd-looking form:

createNews.ftl

We are no longer using standard HTML forms, now we use Spring forms. They are almost identical to HTML ones, but now we also pass some modelAttribute.

That modelAttribute is a parameter that we will pass to and receive form the form. So here we will pass a News entity and when the user will fill in the form we will get the same News entity back but now with filled variables.

You see that there are 3 types of tags inside the form:

label - This is just a label associated with the input tag
input - This can be thought of as a standard HTML input tag (but now we don’t set type ourselves)
error - if some errors will occur that will be returned to the form and will be associated with a certain variable of our News entity.

Path is a parameter that tells to which variable inside our News entity that content is related.

Validating the News Entity

What validation checks do we want?

    • Headline should be not empty
    • Headline should be maximum of 100 characters in length
    • Text should not be empty
    • Headline of each News should be unique

The first three validations can be done without us writing Java code. We only need to connect a new dependency for Hibernate validations. This is the link to the Hibernate validations dependency.

I will be working with Hibernate Validator 6

pom.xml

        <dependency>

            <groupId>org.hibernate.validator</groupId>

            <artifactId>hibernate-validator</artifactId>

            <version>6.0.8.Final</version>

        </dependency>


   <!-- ... -->

Hibernate validator does the following:

When we get an object with annotation @ModelAttribute we can ask Hibernate to validate the fields of that object for things like:

    • If the field is empty or not?
    • What length does it have?
    • Is it in email format?

And this is how it’s done:

    • Next to @ModelAttirbute add @Valid annotation which tells Hibernate to validate some of the fields.
    • Annotate the variables inside the entity class with Hibernate annotations:

Here are some common annotations for Hibernate validations:

@NotBlank - Field should not be empty
@Size - Field should be in some range of characters (it can be set by setting min and max parameters for this annotation)
@Email - Field should be in format of email

After adding these annotations, validation errors (if any occur) will be automatically added to the BindingResult object for an appropriate field.

But we also need another check. We want to see if the headline of the News is not already used by some other News object.

Spring has a special interface that allows us to make more complex checks. This interface is called Validation and we will implement it.

Now, back to our validation. We want to register that an error had occurred if validation had failed. For that we have Errors object in the parameters of method validate().

We can use that object to add errors like this:

NewsValidator.class:


@Component

public class NewsValidator implements Validator {


    // ...


    @Override

    public void validate (Object target, Errors errors) {

        News news = (News) target;

        if (newsService.getNews(news.getHeadline()) != null){

            errors.rejectValue("headline" , "This headline is already in use");

        }

    }

}



We are done creating our validations. We only have to finish writing our method for NewsController:

}
