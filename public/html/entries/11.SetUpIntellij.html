Setting up IDE

Before we begin writing any code we need to do some basic set up.

Downloading IDE

Every project starts with a “Create a Project” button. So, to be able to start writing any code we will need to set up our IDE. Spring is a framework that works with java enterprise edition (or Java EE for short), and we will need IDE that supports Java EE. The choice of IDE for development is a choice that every programmer has to make themself and I cannot tell you which one is better. My preferred one, though, is Intellij Idea. I find it more intuitive to use. Another common option is Eclipse.

One big difference between Intellij and Eclipse is that Eclipse is free to develop Java EE applications and Intellij Idea isn’t. But again, I find Intellij Idea more easy to use.

Here are the links you can follow to download Intellij Idea and Eclipse IDEs

Intellij IDEA:
https://www.jetbrains.com/idea/download/

Note: To work with Spring you will need Intellij IDEA Ultimate.

Eclipse:
https://www.eclipse.org/downloads/

For all of the future examples I will be using Intellij Idea.

Creating simple Java EE project

Step One: Create a basic project from archetype.

The process of initializing a Java EE-based project is almost identical to the creation of any other Java project. However, the easiest way to create a project is by using an archetype. An archetype is a carcass for a project that will be automatically generated when you finish the setup in your IDE. That is especially useful as the program will generate most of required files for you.

Do the following:

    • Select "Maven" option on the left
    • Select “Create from archetype" option on the right
    • Select archetype called “maven-archetype-webapp”

Step Two: creating a folder hierarchy in the project

Right now we only have a few files and a folder called webapp under our main folder. We will not focus on the files that are present in those folders now, but instead we will focus on the folders themselves.

Usually you will have 3 folders under your main folder:

1. Folder webapp is a folder that stores files associated with running the application. In there you can store the following categories of files:

    • Static content like HTML files (those files that do no get updated)
    • JavaScript files
    • CSS files
    • Dynamic content - files that can change their content while the application runs (we will have a look at them in greater depth later)
    • Web.xml file (we will talk about it later)
    • Servlet and config files (we will also talk about them later in the course)
    • Jar files (library files)

2. Folder resources is where you store your properties and configuration files. In our course the only files that we will be storing in this folder are .properties files (we will talk about them a bit later).

3. Last folder is the java folder. Under it you will be storing all your java code.
