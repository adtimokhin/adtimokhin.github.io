MVC

What is MVC

MVC is a programming pattern for creating applications. It is made out of three main components:

M - model

V - view

C - controller

We have already touched on what controllers are, and there is not much to add: they are objects that get a HTTP request from a user and decide what method to call to handle the request.

We already have seen views. Actually, we already have created some. Views are any files that clients receive from our server for them to see and interact with. This can be HTML pages with CSS and JS added to them, or templates (like FTL or thymeleaf).

Model is an object that defines how data is stored in the program. We also use models for storing data in databases.

How MVC works

It is quite simple.

1) Controller gets HTTP request from a user

2) Controller decides what method will handle that particular request

3) That method can manipulate data we got from the user

4) Model is used to send queries and get results from the database

5) This data is used by our controller

6) View is returned to user
