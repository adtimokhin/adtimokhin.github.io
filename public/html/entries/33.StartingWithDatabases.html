Getting started with Databases

What is a database

Database allows you to store data in a convenient way. All data is split into tables. Each table stores some kind of data. That "kind of data" is data that is grouped under some common entity. For example, you can have a table that will store information about your users. Each user should have an id, username and password. So every column will represent a single parameter of an entity. Table for users may look something like this:

Columns store the same type of data (like passwords), rows store one entity (like user).

Most databases make use of a query language named SQL. It allows us to communicate with databases. If you are unfamiliar with SQL, I suggest you spend a few hours getting the basic idea. The language itself is quite easy, and right now you don't need to go into depths of it. There are many wonderful tutorials out there to learn SQL. I can recommend you try this one.

Connecting database to our Spring application

Spring applications can make use of databases. Before starting to work with it though we need to download the database onto our machine. There are many databases to choose from: MySQL, PostgreSQL, Oracle Database and many others. No matter which one you choose - all of them use more or less same syntax.

I will teach you how to work with PostgreSQL. If you prefer any other database I am certain that you will be able to use it in the Spring application online.

Step Three: Adding dependency

To make our Java code be able to communicate with our Postgres database we need a special class called Driver. There is a Driver class for any database that you will be using so you will be able to find it in some dependency.

This one is for postgreSQL

Do the following:

    • Get Maven dependency with Driver class for your database

This is how it will look for Postgres:

pom.xml:

        <dependency>

            <groupId>org.postgresql</groupId>

            <artifactId>postgresql</artifactId>

            <version>42.2.23</version>

        </dependency>

Step Four: Creating .properties file for our database

What we have done in the previous step is we have established a connection with the database so that our IDE can work with our database. But we want our Spring application also to have access to the server as well.

First of all, in order to establish a connection we need to store information about that connection inside our app. There are a few ways of doing this. For example, you can create some final static Strings and use them to store information about the connection.

But there is a better one. We can create a seperate file where we will store a list of properties that we need for establishing our connection. That is considered a better way because it is much simpler to update your data if it changes.

And now we finally get to use the resource folder!

Do the following:

    • Create file persistence.propertiesinside the resource folder.
    • Add required properties inside our persistence.properties file.

Inside the resource folder:

A filetype with extension .properties is a special file type that stores only a list of properties.

In order to establish connection, we are going to need the following properties:

user - that’s your username for the postgres database
password - that’s your password for the postgres database
url - that’s the url that we have copied from the previous step. It tells Spring where is the database
driver - that’s the folder hierarchy where your Driver class can be found.

Step Connecting our java application to our postgres database

Now let’s finally establish the connection!

There is a special class called Connection that will allow us to communicate with our database through code.

We can create that Connection object inside a configuration file where we would also set all the properties we have created.

Do the following:

    • Create a new configuration file where we have a bean of class Connection.

PersistenceConfig.class:

@Configuration

@PropertySource ("classpath:persistence properties")

public class PersistenceConfig {


    @Bean

    public Connection getConnection (@Value ("${user}") String user,

                                                                   @Value ("${password}") String password,

                                                                   @Value ("${url}") String url,

                                                                   @Value ("${driver}") String driver){


        try {

            Class.forName(driver) ;

            return DriverManager.getConnection(url, user, password) ;

        } catch (SQLException | ClassNotFoundException e) {

            return null ;

        }

    }


}


To tell Spring where our .properties file is, we use @PropertySource annotation. As a parameter we set the classpath to our persistence.properties file.

@Value annotation is used to tell what value from the property file we want to download. As a parameter of the annotation we pass a name of the property in the following format:

“${property name}”

Yay! We have established the connection. Yep. It's that simple.

Don’t forget to add this configuration class to our WebAppInitializer!
