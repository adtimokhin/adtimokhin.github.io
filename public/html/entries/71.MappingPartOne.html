Mappings. Part One

Let’s take a break from Spring Security and talk about something new.

I want to implement a new feature that will be used with our models. I want to map my User objects to my News. I want to make it so that every News has a User that has created it and that all information about a single User can be accessed if you have a News object. Create associations between User and News objects, in other words.

That is where the concept of foreign keys comes into play.

Foreign Keys

A foreign key is an attribute of an object inside a table that links that object to another table.

We want to have a User object associated with a single News object. We can create a column inside the “news” table named “author_user_id”. In that column we will store the values that also appear inside the “users” table in the “id” column.

A foreign key in that case will be the “author_user_id” column as it maps to the content inside another table (“users”).

It is possible to create a table that has multiple foreign keys, or which is totally made out of the foreign keys.

So, let’s create the foreign key for our “news” table.

Do the following:

    • Create a new column “author_user_id” in the “news” table. It should have the same type of data stored in it as “id” column in “users” table
    • Make the “author_user_id” column be a foreign key that is linked to the “id” column in the “users” table.

Next stage is to map entities.

Mapping Entities

Of course, the easiest way to map User and News entities will be by simply setting the value of the “author_user_id” manually. But there will probably be a point where we would want to get some more information about the user who had created a particular news. In that case we will need to use UserService to find the User and then get required data of that entity.

This is what essentially is done when we are using mappings. The only difference is that we will be able to get a User that a particular News object has as soon as we get that entity from the database.

There are 3 types of mappings:

One to One

That is only one User can be mapped to one News, and one News can be mapped to only one User

One to Many

That is one User can be mapped to many News

Many to Many

That is many User objects can be mapped to many News and vice versa.

We will be using One to Many mapping because every user can be connected to as many News articles as they want but only one user can be connected to each News.

Important thing to keep in mind: One to Many and Many to One mappings come together. In our example, there is One to Many mapping for User objects and Many to One mapping for News (because many news can only have one user linked to them).

The hardest thing about mappings is to understand what mapping to use.

It is very simple to implement One to Many and Many to One mapping when working with ORM or JPA. We just need to make use of a few annotations.

What essentially happens is that every News will have a User entity linked to them, so we need to create a User inside our News model:

News.class:

@Entity

@Table (name = "news")

public class News {


    // ...


    private User user;


    // ...

}

And now adding required mapping annotations:

News.class:

@Entity

@Table (name = "news")

public class News {


    // ...


    @ManyToOne

    @JoinColumn (name = "author_user_id")

    private User user;


    // ...

}

@ManyToOne annotation tells Spring that this is the Many To One mapping that is being used in that object.

@JoinColumn annotation tells us what is the column that we use to put the result of mapping one entity to other into (in this case, where we will store the id of User objects)

We need to add getters and setters for this User:

News.class:

@Entity

@Table (name = "news")

public class News {


    // ...


    // getters and setters

    public User getUser () {

        return user;

    }


    public void setUser (User user) {

        this.user= user;

    }


    // ...

}

Now we need to do almost the same thing in the second entity of the relation - User.

We have to create a list of News that User is attached to (we use List because this is One to Many mapping and one User can be linked to many News):

User.class:

@Entity

@Table (name = "users")

public class User {


    // ...


    private List<News> news;


    // ...

}

We need only one annotation here:

User.class:

@Entity

@Table (name = "users")

public class User {


    // ...

    

    @OneToMany (mappedBy = "user")

    private List<News> news;


    // ...

}

@OneToMany annotation signals that in this entity we have One to Many mapping. When working with ORM we engage with objects rather than actual fields of the table. Because of that we need to tell Hibernate what field is used to represent the relation in the table with the foreign key. It is done by setting the mappedBy parameter to the name of the attribute used in the related class. In this case, this name is “user”, because that is what we’ve called the User object inside the News class.

We don’t use @JoinColumn annotation here because the foreign key column for this mapping is not in the table related to this entity (i.e. “users”).

Don’t forget getters and setters for this list as well:

User.class:

@Entity

@Table (name = "users")

public class User {


    // ...


    // getters and setters


    public List<News> getNews () {

        return news;

    }


    public void setNews (List<News> news) {

        this.news= news;

    }


    // ...

}

Believe it or not but we have completed the mapping. Next step is to understand how we work with mappings inside our services.

We will focus on that, as well as working with other mappings, on the next page.
