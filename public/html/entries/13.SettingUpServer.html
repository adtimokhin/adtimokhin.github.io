Setting up a Tomcat Server for our application

What is a server?

Spring Framework is a framework that is used to create the back-end software. Back-end is code which receives some data from a client computer and does some work with it. That “work” may very well be putting that data into a database, or logging user onto a portal. But handling the requests from the internet is not done directly by the code that we will run. It will be done by a server. Server is a computer that handles requests of other computers (called clients).

Besides handling requests, servers incorporate logic that allows them to interact with client computers. This logic includes protocols of receiving information from other computers and protocols for sending that information back to them.

We do not need to worry how servers handle their input and output data. We, as back-end developers, focus on how to handle that data (which is essentially the back-end).

To be able to focus on the back-end we will install some specialized software which will help to turn our machine into a proper server.

What steps to take:

Step One: Install Apache Tomcat to your machine

We will be using the Apache Tomcat server. To install the server follow this link:

https://tomcat.apache.org/download-10.cgi

The newest version there is when I write these words is Tomcat 10.

But beware that older versions of Intellij IDEA (like my one) do not support the newest Tomcat 10 yet. If you can, use Tomcat 10, but if you run into problems you are more than welcome to switch to Tomcat 9. It works as well.

To make sure which file to download you can read the official README file

After you download the file, unzip it (if it is zipped) in your folder with servers (it can be any folder you like).

Step Two: Configure your Spring application to work with Tomcat

In Intellij Idea you will see the Add configuration button at the top of the screen.

Press it.

Click the + button and select the Tomcat option from the dropped list.

Select the local option as you have installed your Tomcat server onto your computer.

On your screen you will see a window which looks something like this:

Step Three: Set up basic configuration for Tomcat to run

Next to the box where it says Application server: you will probably see nothing. We need to add our Tomcat there.

Press Configure button, and you will see a screen with a list of your local Tomcat servers. This is how mine looks:

You will probably see no servers on your screen, so we have to add one!

Press + button on the left of the screen

In the window that pops up specify the folder with all the Tomcat files that you have downloaded in the first step.

As I have said, my Intellij IDEA doesn’t support Tomcat 10, so I will be using Tomcat 9.

Press Configure button and now you have configured the Tomcat server.

The last step we need to make it all work is to fix an error that we see at the bottom:

Press fix button and select the option where it says war exploded

Don’t forget to press the Apply button.

Voila! You have configured the server! Step Five: Troubleshooting

When you try to run your code you may get this error:

This error happens because we didn’t grant the rights to execute a file called catalina.sh in the folder with our Tomcat server.

I will explain how to solve this issue on Mac OS.

To fix it go to terminal and type the following:

1) Go to the folder with your Tomcat server

In command line write: cd (directory path)

2) Go to the bin folder

In command line write: cd bin

3) Grand the rights to execute catalina.sh file

In command line write: chmod +x catalina.sh
